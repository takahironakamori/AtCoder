### ABC008

# C - コイン

  [問題はこちら](https://atcoder.jp/contests/abc008/tasks/abc008_3)

- 発想<br>
  期待値：確率変数が取る値を、確率によって重み付けした平均値。<br>
  2, 4, 8 の 3 つのコインで考えると、以下の 6 パターンになる。<br>
  2白 4黒 8白 で白 2 個<br>
  4白 2白 8白 で白 3 個<br>
  2白 8黒 4黒 で白 1 個<br>
  4白 8黒 2白 で白 2 個<br>
  8白 2白 4黒 で白 2 個<br>
  8白 4白 2白 で白 3 個<br>
  白の期待値は 13/6 となる。<br>
  しかし、各コインですべてのパターンを洗い出して、白を数えると計算量は O(N!) となる。<br>
  O(N!) だと N は 10 くらいまでしか間に合わない。(10 の階乗は 3,628,800)<br>

  パターンごとに調べると計算量が多いので、コインごとに見ていく。<br>
  2 を見た場合、白は 6 個なので、期待値は 1/1<br>
  4 を見た場合、白は 3 個なので、期待値は 1/2<br>
  8 を見た場合、白は 4 個なので、期待値は 4/6<br>
  白の期待値の合計は、13/6 となる。<br>

  2 を見た場合、2 の約数はないので、約数が 2 のコインより左側に来ることはありえないので、1 パターン、白になる。<br>
  4 を見た場合、4 の約数は 2 の 1 つ。<br>
  2 と 4 だけで考えると、<br>
  2白 4黒<br>
  4白 2白<br>
  なので、白は 1 個、パターンは 2 なので、期待値は 1 / 2。<br>
  すなわち、期待値は、全パターンのうち奇数番目に来る確率に等しい。<br>
  言い換えると、全パターンのうち自身のコイン(4)より左側にいるコインが偶数枚ある場合の確率であり、<br>
  この場合は、今回の 4 のように 1 / 2になる。
  8 を見た場合、8 の約数は 2 と 4 の 2 つ。<br>
  パターンは上記と同じ 6 パターンなので、期待値は 4 / 6。<br>
  すなわち、全パターンのうち奇数番目に来る確率に等しい。<br>
  言い換えると、全パターンのうち自身のコインより左側にいるコインが奇数枚ある場合の確率であり、<br>
  自身以外のコインの枚数を S をすると (S + 2) / (s × 2 + 2) で計算できる。
  上記の 8 で考えると (2 + 2) / (2 × 2 + 2) = 4 / 6。

- コード（C++)

  ```cpp
  #include <bits/stdc++.h>
  using namespace std;

  int main() {

    int N;
    cin >> N;

    vector<int> C(N);
    for (int i = 0; i < N; i++) {
      cin >> C[i];
    }

    sort(C.begin(), C.end());

    double answer = 0.0;

    for (int i = 0; i < N; i++) {
      int count = 0;
      for (int j = 0; j < N; j++) {
        if (i != j && C[i] % C[j] == 0) {
          count++;
        }
      }
      if (count % 2 == 0) {
        answer += (double) (count + 2) / (count * 2 + 2);
      } else {
        answer += 0.5;
      }
    }

    cout << setprecision(11) << answer << endl;

    return 0;

  }
  ```
